# 1.0 — Native runtime state-ownership design (before Phase-1 code)

The concurrency design is a DESIGN, not a checker (TSan/ASan are secondary). It must preserve, in a multi-thread C++
runtime, the invariants the Python server enforces with `state_lock` + sorted lock acquisition + generation counters +
fork isolation. Grounded in: `server.py` `state_lock` :486-490, sorted lock acq :5086-5089/:5190-5192, generation gates
:5201-5225/:6827-6842/:7660-7679, fork :6370-6425 + FORK_ASSERT :6792-6797, hyp-aliasing guard `batch_primitives.py`
:109-126.

## Core model: per-session ownership + immutable-snapshot → result-commit
1. **`Session` owns all its mutable state** — encoder cache (`cache_last_channel/_time/_channel_len`), decoder
   `Hypothesis`/`LabelLoopingStateItem`, audio rings (raw_audio_ring, mel_frame_ring, pending_audio), the finalize state
   machine, and a monotonic **generation** counter. **No lane thread ever mutates `Session` state directly.**
2. **Batched GPU+decode works on OWNED batch buffers, not session state.** The scheduler, holding the scheduler mutex,
   assembles a batch of ready chunks; for each it **snapshots** the session's cache + decoder state into the batch's
   working buffers (the *scatter* — port `batch_primitives.stack_*`). A lane thread runs preprocess → encoder graph
   replay → native decode entirely on those buffers. **The single point of session-state mutation is the *commit*** (the
   *scatter-back*), done back under the session's guard.
3. **Hypothesis aliasing is forbidden.** Each batch row gets its OWN deep-copied `Hypothesis`/`LabelLoopingStateItem`
   (NeMo mutates hyps in place — `batch_primitives.py:109-126`). The commit writes back per-row; no shared hyp object
   crosses rows or threads.

## Generation / stale-output rules (replicate the Python suppression)
- Each `Session` has a `generation` (monotonic). **`vad_stop`, `reset`, `cold-reset` increment it** (mirror
  :5755-5770/:5907-5928).
- A batch result carries the generation it was snapshotted at. **At commit/emit: if `session.generation` changed since
  the snapshot, DISCARD the result** (stale) — no commit, no emit. This reproduces the generation-gated emit/finalize
  suppression (:5201-5225, :6827-6842, :7660-7679). Prevents a stale steady/finalize result landing after a newer reset.

## Finalize fork isolation
- Finalize **snapshots** (deep-copies) the session's decode state into a *fork*, runs padded-finalize encode
  (keep_all_outputs T_max) + final decode on the fork, computes the final delta, emits it — and the **parent session
  state is NOT mutated** (so a resumed utterance continues correctly). Mirror :6370-6425; port the **FORK_ASSERT**
  parent-unchanged check (:6792-6797) as a debug-gated invariant.
- The fork result is committed ONLY as the emitted final (+ the session transitions FINALIZED/cold-reset under its
  generation guard); the fork's mutated decode state is discarded.

## Finalize state machine (owner-serialized)
`STREAMING → PENDING_FINALIZE → FINALIZED` (+ resume), owned and transitioned ONLY by the session's context:
- `vad_stop` → invalidate generation, arm debounce, PENDING_FINALIZE (:5755-5770).
- audio during PENDING_FINALIZE → held in `post_stop_audio`, NOT appended (:5590-5598).
- `vad_start` before debounce → cancel debounce, flush held audio back (:5731-5744).
- `reset` before debounce → set `reset_seen`, wait for debounce (:5823-5829).
- `end`/`close` → force finalize (include/discard held audio) (:5676-5684/:5802-5815).
The T1 trace suite (random interleavings) validates exact event order + post-stop byte counts + generation transitions.

## Threading & locks (fine-grained; NOT the coarse global lock)
- **Scheduler mutex**: guards the ready-set, lane assignment, batch assembly. Short critical sections.
- **Per-session guard**: a lightweight per-session lock (or single-owner actor execution) around snapshot + commit +
  state-machine transitions. Replaces the coarse `asyncio.Lock`/exclusive-gate that serialized everything.
- **Lane threads**: own a CUDA stream + captured graph + batch working buffers; do GPU + decode with NO session-state
  access (only the snapshot they were handed) → finalize overlaps steady across lanes (the density win).
- **No global `inference_lock`**: removing it is the point. Correctness comes from snapshot/commit + generation guards,
  not a global serialization.

## Enforcement
C++ discipline (we chose all-C++; the borrow-checker rationale doesn't apply): the snapshot/commit are the ONLY
cross-thread touch points and are guarded; everything else operates on owned buffers. **CI: TSan + ASan** on the
concurrency tests, plus the **per-session sequencing canary at load** (the real correctness gate — TSan proves no data
race, NOT logical ordering; the canary proves no stale/dup/reordered emit).

## Open design choices (decide in implementation)
- **Actor (single thread per session, message-passed) vs lock-guarded shared `Session`.** Lean: lock-guarded shared
  Session + the scatter/commit protocol (lower overhead than per-session threads at 40+ sessions); actors if the
  lock-guarded version shows contention.
- **Snapshot cost**: the cache deep-copy per batch row per chunk must be cheap (it's on the hot path) — measure; the
  encoder graph already needs the cache copied into static buffers, so the snapshot ≈ that copy.
