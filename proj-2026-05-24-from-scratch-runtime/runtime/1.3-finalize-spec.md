# 1.3 — Continuous-finalize spec (extracted from server.py for the native-runtime port)

Source: `src/nemotron_speech/server.py` (NEMOTRON_CONTINUOUS=1 path). This is the CORE single-stream algorithm with all
production scaffolding (asyncio scheduler, batching, CUDA-graph capture, profiling, priority lanes, websocket I/O,
metrics) deliberately stripped. Cite these line anchors when porting/reviewing.

## 1. State machine — `continuous_state`
Values (server.py:91-93): `STREAMING`, `PENDING_FINALIZE`, `FINALIZED`. Field default `STREAMING` (489).
Transitions:
- **STREAMING → PENDING_FINALIZE** on `vad_stop` (`_scheduler_continuous_handle_vad_stop_locked`, ~6858-6877; assign 6866). Starts a debounce timer = `finalize_silence_ms` (default 150ms, line 86); `continuous_stop_seq++`.
- **PENDING_FINALIZE → STREAMING** on `vad_start` before debounce expires (~6830-6856; assign 6836). Cancels debounce; flushes `continuous_post_stop_audio` back to pending (6844). (This is what makes finalize SPECULATIVE → why the fork must not mutate the parent.)
- **PENDING_FINALIZE → FINALIZED** on debounce expiry (`_..._handle_debounce_expired_locked`, ~6957-6994; assign 6978) OR immediate hard reset on `end`/`close` (`_..._force_finalize_locked`, ~6766-6808; assign 6800).
- **FINALIZED → STREAMING** speculative finish (context retained) `_continuous_finish_speculative_finalize_locked` (~9014-9035) OR true-boundary cold reset `_continuous_cold_reset_after_finalize_locked` (~9079-9121).

## 2. Finalize encoder pass (the remainder) — the DEFINING difference: `keep_all_outputs=True`
At finalize, the held remainder audio is turned into mel and run through ONE encoder step with **`keep_all_outputs=True`**
(steady uses `False`). keep_all_outputs=True emits ALL encoder frames including the right-context/look-ahead tail that the
steady path withholds — correct because no more audio is coming. This is the accuracy point of finalize.
- Remainder frame math (~8009-8013): `padded_total = emitted_frames*hop + len(pending_audio)`; `total_mel = padded_total//hop + 1`; `remaining = total_mel - emitted_frames`.
- Chunk composition (~8070-8076): if `emitted_frames==0`: `chunk_mel = new_mel`, `drop_extra=0`; else `chunk_mel = cat(mel_frame_ring, new_mel)`, `drop_extra = self.drop_extra` (=2).
- Final-padding: fork appends `final_padding_frames = (right_context+1)*shift_frames` of silence to pending before the pass (7487-7488).
- Call (serial path `_process_final_chunk`, ~9988-10000): `_conformer_stream_step(processed_signal=chunk_mel, processed_signal_length=chunk_len, cache_last_channel/time/channel_len=<cloned>, keep_all_outputs=True, previous_hypotheses=<cloned>, previous_pred_out=<cloned>, drop_extra_pre_encoded={0|2}, return_transcription=True)`.
- Geometry: first chunk T=remaining; continuation T = pre_encode_cache_size(9) + remaining.

## 3. Fork / clone isolation (`NEMOTRON_FORK_ASSERT`)
`_build_continuous_finalize_fork` (~7473-7605) deep-clones BEFORE finalize so the live stream is untouched if finalize is
cancelled/speculative:
- encoder caches: `cache_last_channel/time/channel_len` via `tensor_clone` (=`.detach().clone()`, 149) (7507-7521)
- decoder: `previous_hypotheses = clone_hypotheses_deep(...)` (212), `pred_out_stream = clone_tree(...)` (157) (7522-7523)
- `mel_frame_ring = clone_tree(...)`, `emitted_frames` copied (7505)
- text state: current_text/last_emitted_text/committed_text/continuous_emitted_text (7524-7527)
- audio: pending_audio.copy(), raw_audio_ring.copy() (7491-7521)
FORK_ASSERT (env, 623): snapshot parent before (`_snapshot_fork_assert_parent`, 7607-7626 — clones caches+hyps+pred_out),
and after the finalize flush assert parent BYTE-IDENTICAL (`_assert_fork_flush_parent_unchanged`, 7628-7668; `_assert_tree_equal`). The native port's FORK_ASSERT must do the same byte-equality check.

## 4. Decode continuation across finalize
The cloned `previous_hypotheses` + `pred_out_stream` (the RNNT pred-net hidden state) are passed into the finalize encoder
step so the finalize decode CONTINUES from the streaming decode state (partial-hypotheses continuation), not from scratch.
final_text extracted from the returned transcription (~10016-10025).

## 5. Reset / resume
- Speculative finish (context retained, ~9014-9035): set STREAMING; clear debounce flags + `continuous_reset_seen`; KEEP committed_text, continuous_emitted_text, encoder caches, decoder state, mel_frame_ring, emitted_frames, raw_audio_ring (they advance from the fork's final state IF committed).
- True-boundary cold reset (~9079-9121): clear committed_text/last_emitted_text/continuous_emitted_text/overlap_buffer; `continuous_post_stop_audio.clear()`; `_init_session()` (resets caches, decoder state, mel_frame_ring); then STREAMING.
- Delta/dedup emission (`_continuous_append_only_delta`, ~392-417): emit only the new word-suffix vs `continuous_emitted_text`; suppress deletions/corrections; update committed_text + continuous_emitted_text after emit (7948-7970).

## 6. Constants (NeMo streaming cfg, ~1581-1647)
shift_frames=16 (~160ms), pre_encode_cache_size=9 (~90ms), drop_extra=2, BLANK=1024, MAX_SYMBOLS=10,
final_padding_frames=(right_context+1)*shift_frames, FINALIZE_SILENCE_MS default 150. steady T = 9+16 = 25.

## Native-port correctness anchors
- T1 finalize canary: streaming-steady + keep_all_outputs=True finalize remainder == NeMo's reference complete decode
  (the finalize recovers the right-context tail the steady path withholds).
- FORK_ASSERT: parent caches + previous_hypotheses + pred_out_stream byte-identical after a finalize.
- reset/resume: multiple concatenated turns; speculative (context retained) vs cold (true boundary) both correct.
