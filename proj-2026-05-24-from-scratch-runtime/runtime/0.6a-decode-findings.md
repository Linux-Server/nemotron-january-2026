The funding-gate sub-problem: a native `greedy_batch` LABEL-looping RNNT decode that reproduces NeMo BYTE/STATE-exact.
This doc captures the **contract** + **golden fixtures** + the **plan to a verified reference then C++**.

## The decode contract (captured live вЂ” `capture_decode_fixtures.py`)
Deployed config (confirmed from `change_decoding_strategy`): **`RNNTBPEDecoding` + `GreedyBatchedRNNTInfer`**,
`strategy=greedy_batch`, `loop_labels=True`, `use_cuda_graph_decoder=False`, `max_symbols(_per_step)=10`. Tokenizer =
SentencePiece, 1024 tokens.

**Input:** `enc [B, 1024, T]` (encoder output, channel-first), `enc_len [B]`.
**Output:** `List[Hypothesis]`, fields: `score, y_sequence (int64 token ids), text, dec_out, dec_state, timestamp,
alignments, ..., y, lm_state, ngram_lm_state, tokens, last_token, token_duration, last_frame`.
**`dec_state = LabelLoopingStateItem`**, fields: **`predictor_state, predictor_output, label, decoded_length, lm_state,
time_jump`** вЂ” for our RNNT EN path: **`lm_state=None`, `time_jump=None`** (TDT-only). The native decode must carry
`predictor_state` (LSTM h,c), `predictor_output`, `label` (last token), `decoded_length`.

## Algorithm to reproduce (label-looping greedy, RNNT)
Per `rnnt_label_looping.py`: loop over labels, advancing time t when blank; force blank/time advance after `max_symbols`
non-blank per frame (`:466-484`); carry predictor LSTM state + last label + decoded_length; pack `y_sequence` and move
state to CPU at the end (`rnnt_greedy_decoding.py:52-86`). Modules: `model.decoder` (prediction net, LSTM) +
`model.joint` (joint net) вЂ” both linear/LSTM, so libtorch with the SAME weights should match the math; the risk is the
LOOP logic + arg-tie-breaking + state handling, not the GEMMs.
