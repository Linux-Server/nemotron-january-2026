# 0.6a — native decode equivalence: findings + golden fixtures (foundation)

The funding-gate sub-problem: a native `greedy_batch` LABEL-looping RNNT decode that reproduces NeMo BYTE/STATE-exact.
This doc captures the **contract** + **golden fixtures** + the **plan to a verified reference then C++**.

## The decode contract (captured live — `capture_decode_fixtures.py`)
Deployed config (confirmed from `change_decoding_strategy`): **`RNNTBPEDecoding` + `GreedyBatchedRNNTInfer`**,
`strategy=greedy_batch`, `loop_labels=True`, `use_cuda_graph_decoder=False`, `max_symbols(_per_step)=10`. Tokenizer =
SentencePiece, 1024 tokens.

**Input:** `enc [B, 1024, T]` (encoder output, channel-first), `enc_len [B]`.
**Output:** `List[Hypothesis]`, fields: `score, y_sequence (int64 token ids), text, dec_out, dec_state, timestamp,
alignments, ..., y, lm_state, ngram_lm_state, tokens, last_token, token_duration, last_frame`.
**`dec_state = LabelLoopingStateItem`**, fields: **`predictor_state, predictor_output, label, decoded_length, lm_state,
time_jump`** — for our RNNT EN path: **`lm_state=None`, `time_jump=None`** (TDT-only). The native decode must carry
`predictor_state` (LSTM h,c), `predictor_output`, `label` (last token), `decoded_length`.

## Golden fixtures (`runtime/fixtures/`, committed — small)
| fixture | enc | tokens | text |
|---|---|---:|---|
| decode_real_1 | [1,1024,41] | 12 | "How much juice is in one lime?" (EXACT vs ref) |
| decode_real_0 | [1,1024,44] | 20 | "This is Samantha Lee with a night to remember." |
| decode_real_2 | [1,1024,162] | 74 | long sentence (solar panels) |
| decode_noise_3s / _loud / quiet_1s | [1,1024,39/14] | 0 | "" (all-blank edge case) |
A native decode fed the same `enc`/`enc_len` MUST produce the same `y_sequence` (and matching `text`, decoded_length,
predictor state). These exercise: multi-token label-looping, blank handling, a long sequence, and all-blank.
**Missing fixtures (follow-up):** `max_symbols=10` saturation case; streaming **partial_hypotheses continuation**
(2-chunk: state carried across calls) — the deployed streaming path; and per-token `predictor_state` snapshots.

## Algorithm to reproduce (label-looping greedy, RNNT)
Per `rnnt_label_looping.py`: loop over labels, advancing time t when blank; force blank/time advance after `max_symbols`
non-blank per frame (`:466-484`); carry predictor LSTM state + last label + decoded_length; pack `y_sequence` and move
state to CPU at the end (`rnnt_greedy_decoding.py:52-86`). Modules: `model.decoder` (prediction net, LSTM) +
`model.joint` (joint net) — both linear/LSTM, so libtorch with the SAME weights should match the math; the risk is the
LOOP logic + arg-tie-breaking + state handling, not the GEMMs.

## Plan to close 0.6a (de-risked path)

   `model.joint` directly (NOT NeMo's decode), validate it produces **byte-identical `y_sequence`** to NeMo on every
   fixture. This PROVES we understand the algorithm exactly → becomes the executable spec for the C++ port. Locally

   byte-exact, the C++ port can't either — surface that before funding the C++ build.**
