[BLOCKER] `drop=0` first-finalize is a real uncovered geometry, not a fixture oddity. `runtime/build_range_examples.py:14` hard-codes `DROP = 2` and `T_LO,T_HI = 43,58`; `runtime/finalize_t_distribution.py:64` only appends full corpus utterances, drains steady, then stops, so it never intentionally creates a first-finalize-before-steady case. Production sets finalize `drop_extra = 0` whenever `session.emitted_frames == 0` (`src/nemotron_speech/server.py:8071`, `src/nemotron_speech/server.py:8317`, `src/nemotron_speech/server.py:9951`). The first steady chunk is only emitted once the session reaches `(emitted_frames + shift_frames + 1) * hop_samples` (`src/nemotron_speech/server.py:9159`), and final padding adds `(right_context + 1) * shift_frames` frames (`src/nemotron_speech/server.py:1603`, `src/nemotron_speech/server.py:1607`, `src/nemotron_speech/server.py:7486`). With the current English right-context=1 geometry, a short utterance finalized before the first steady emit has exact-T roughly `floor((real_pending + 5120) / 160) + 1`, so non-empty first-finalize spans T=33..49, not just T49. `finalize_ref.prepare_finalize_inputs` implements the same math (`runtime/finalize_ref.py:716`, `runtime/finalize_ref.py:747`). The single `drop=0,T=49` fixture bucket only covers the upper edge. Concrete fix: export and validate `drop=0` exact-T buckets for the full short-utterance range, or add an explicit eager/fallback path for all `drop=0` and prove it in the C++ runtime; add a synthetic short-audio sweep to the corpus probe and shadow gate.

[BLOCKER] Missing or out-of-range buckets do not fail the Python corpus gate. `BucketManager.get()` returns `None` on a missing `(drop,T)` (`runtime/finalize_corpus_shadow.py:136`), `run_sample()` records `covered=False` and `detail="missing bucket"` (`runtime/finalize_corpus_shadow.py:300`, `runtime/finalize_corpus_shadow.py:311`), but the process pass criterion is only `len(divergent) == 0` (`runtime/finalize_corpus_shadow.py:452`) and it returns success even with uncovered rows (`runtime/finalize_corpus_shadow.py:478`, `runtime/finalize_corpus_shadow.py:483`). Concrete fix: make `missing_bucket` and unexpected `no_inputs` fail by default; require `covered == len(results)` unless a named fallback path is exercised and validated.

3. [BLOCKER] The C++ Phase B harness also silently skips missing buckets. The route is exact `(drop_extra, chunk_T)` (`runtime/cpp/finalize_main.cpp:411`, `runtime/cpp/finalize_main.cpp:415`), but if no loader exists it prints "skipped" and continues without marking `phase_b_ok = false` (`runtime/cpp/finalize_main.cpp:416`). The final status can still be PASS (`runtime/cpp/finalize_main.cpp:452`). That means a future fixture row with `drop=0,T=40` or `drop=2,T=59` can produce a green C++ run without testing the bucket path. Concrete fix: treat any missing Phase B bucket as a hard failure, or route to an explicit validated fallback and report it as fallback, not pass.

4. [MAJOR] The T-range claim is an implicit right-context=1 invariant, not an artifact contract. The server accepts `--right-context` values `[0, 1, 3, 6, 13]` (`src/nemotron_speech/server.py:10106`), prompted models default to right-context 3 (`src/nemotron_speech/server.py:1331`), and final padding depends directly on `self.right_context` (`src/nemotron_speech/server.py:1607`). `finalize_ref` hardcodes `[70, 1]` (`runtime/finalize_ref.py:189`), and the bucket builder hardcodes only `drop=2,T=43..58` (`runtime/build_range_examples.py:14`). Large post-stop audio during debounce is probably not the reason for T>58 because PENDING audio is held outside `pending_audio` (`src/nemotron_speech/server.py:7207`) and true-boundary `end/close` flushes it through the steady drain before finalizing (`src/nemotron_speech/server.py:7280`, `src/nemotron_speech/server.py:9153`). But that bound is nowhere asserted in the AOTI artifacts. Concrete fix: embed and check model id, att-context, right-context, shift, pre-cache, final-padding, drop, and supported `(drop,T)` ranges in the bucket manifest and C++ startup; refuse to run buckets outside that contract.

Concrete fix: either run the same 1000 finalized rows against the NeMo stream+finalize oracle, or downgrade the claim to "AOTI buckets match eager finalize_ref on observed drop=2 corpus rows."
