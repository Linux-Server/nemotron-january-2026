# Adversarial Review: Phase-1 1.4 C++ Single-Session Exit Gate

1. [BLOCKER] The gate validates cumulative token end-state, not the T1 behavioral contract.
   - Evidence: the T1 ship gate requires "full-1000 WER", per-utterance WER bounds, and "exact per-session event-stream / final-delta / generation-suppression equivalence" (`reviews/0.4-decision-FINAL.md:23-26`). The C++ harness states its narrower contract up front: "MEL-level session composition" and "emitted cumulative tokens must exactly equal finalize_ref gold tokens" (`runtime/cpp/session_main.cpp:1-6`). The actual assertions are vector equality only (`runtime/cpp/session_main.cpp:159-178`, `runtime/cpp/session_main.cpp:985-986`, `runtime/cpp/session_main.cpp:914`, `runtime/cpp/session_main.cpp:1011-1016`).
   - Miss: a runtime can emit wrong partials, wrong final deltas, duplicate finals, stale finals, or no websocket events at all and still pass 200/200 as long as the final cumulative token vector matches.
   - Concrete fix: promote 1.4 from "token replay" to an event-stream oracle: capture ordered server events for each session, including `type`, `text`, `is_final`, `finalize`, suppression reason, and cumulative collector state, then compare C++ output event-for-event against the production server path. Keep cumulative tokens as one invariant, not the exit gate.

2. [BLOCKER] Final-delta equivalence is not checked; the exported `finalize_new_tokens` is not the production final delta and is unused by C++.
   - Evidence: the exporter stores cumulative steady tokens, cumulative final tokens, and a token suffix `final_tokens[len(steady_tokens):]` (`runtime/export_session_bundle.py:131-145`), then registers that suffix as `utt{i}_finalize_new_tokens` (`runtime/export_session_bundle.py:198-201`). The C++ final path ignores it: it reads only `gold_tokens` (`runtime/cpp/session_main.cpp:877-880`) and compares `fork.hyp` to that cumulative gold (`runtime/cpp/session_main.cpp:912-915`).
   - Production behavior is text-delta based, not "tokens after steady". The append-only delta function works against `session.continuous_emitted_text` (`src/nemotron_speech/server.py:392-417`) and the final emit path sends only `delta_text` when non-empty (`src/nemotron_speech/server.py:8936-8983`), otherwise suppresses empty/duplicate finals (`src/nemotron_speech/server.py:8998-9012`).
   - Miss: for the first speculative finalize, production delta is the collector-safe suffix relative to previous final emissions, often the whole final text, not just tokens newly recovered after steady decode. For later turns it is relative to prior speculative final text, not the current utterance's steady prefix. The current gate cannot catch a broken `_continuous_append_only_delta` port.
   - Concrete fix: export and compare production-style text deltas and collector text after every final event. Add adversarial cases where `final_text` is a non-prefix correction of the already emitted collector, a duplicate final, and a shortened final.

3. [BLOCKER] Generation-suppression equivalence is entirely absent.
   - Evidence: production finalize has multiple stale-generation suppression points before enqueue/model work and after model work (`src/nemotron_speech/server.py:8694-8729`, `src/nemotron_speech/server.py:8797-8815`, `src/nemotron_speech/server.py:8882-8900`, `src/nemotron_speech/server.py:8941-8959`). Generation invalidation is a real state transition (`src/nemotron_speech/server.py:6589-6602`). C++ `SessionState` has no scheduler generation, no queued finalize, no stale event, and no suppression state (`runtime/cpp/session_main.cpp:46-57`).
   - Miss: a native server can emit a stale final after newer audio has advanced the session and still pass this C++ replay. That is explicitly outside the T1 event/generation bar.
   - Concrete fix: add a scheduled-final replay mode with `expected_generation`, interleaved audio advancement, and a stale finalize attempt. The expected event stream must include no transcript emit and a suppression record matching production's stale-generation reasons.

4. [BLOCKER] Partial/interim event-stream equivalence is not exercised.
   - Evidence: production emits interim websocket transcript events whenever streaming text changes (`src/nemotron_speech/server.py:9165-9177`). The C++ harness only prints local pass/fail summaries (`runtime/cpp/session_main.cpp:1002-1008`) and never models a websocket event queue, client collector, or JSON transcript schema.
   - Miss: the C++ runtime could output cumulative partials at different chunk boundaries, skip interim updates, emit deltas instead of cumulative interim text, or regress text ordering, and 1.4 would not see it.
   - Concrete fix: record per-chunk interim events from the production server for the same audio and compare the complete ordered transcript event stream, not just `steady_tokens` after all steady chunks.

5. [BLOCKER] This is a mel-stream gate, not a native audio-stream gate.
   - Evidence: the exporter says the C++ harness replays a deterministic "MEL-level" bundle "without needing NeMo or an audio frontend in the C++ process" (`runtime/export_session_bundle.py:2-6`). It records Python-produced `new_mel` chunks (`runtime/export_session_bundle.py:63-90`) and Python-produced `final_chunk_mel` (`runtime/export_session_bundle.py:120-123`). C++ consumes those tensors directly (`runtime/cpp/session_main.cpp:826`, `runtime/cpp/session_main.cpp:882`).
   - Production builds fixed-shape audio inputs from raw-ring + align pad + new audio (`src/nemotron_speech/server.py:4518-4567`), feeds audio into the preprocessor, emits partials from audio arrival (`src/nemotron_speech/server.py:9153-9177`), and assembles final remainder mel by iterating over pending audio and raw-ring state (`src/nemotron_speech/server.py:9903-9932`).
   - Miss: C++ audio->mel preprocessing, raw-ring maintenance, STFT boundary context, `valid_samples`, sample-to-mel alignment, final padding, and final remainder assembly are all trusted from Python. Calling this "single native stream" overstates the evidence; it is a single native mel stream.
   - Concrete fix: make the exit gate audio-fed. Feed PCM to C++, run the native preprocessor/remainder assembler, compare per-chunk mel hashes and final `chunk_mel` geometry against Python, then run the same token/event assertions.

6. [MAJOR] Finalize geometry is pre-baked by the bundle; C++ does not recompute the production remainder math.
   - Evidence: the bundle exports `final_remaining_frames` and `final_padded_total_samples` (`runtime/export_session_bundle.py:149-150`, `runtime/export_session_bundle.py:206-212`), but C++ never validates those fields. It only checks `final_chunk_mel.size(2) == final_T` and `drop_extra` from parent emitted state (`runtime/cpp/session_main.cpp:881-887`). It also ignores the exported `first_preprocess_mel_frame` and `hop_samples`: the bundle writes them into `meta` (`runtime/export_session_bundle.py:165-179`), while C++ validates only indices 1..7 (`runtime/cpp/session_main.cpp:920-938`).
   - Miss: an off-by-one in sample-to-mel finalization, wrong hop accounting, wrong first mel frame, or wrong padded total can be hidden inside the exported mel tensor and still pass.
   - Concrete fix: in C++, recompute `padded_total_samples`, `total_mel_frames`, `remaining_frames`, and `final_T` from raw audio/session counters; assert them against the bundle. Then delete the mel shortcut from the exit gate.

7. [BLOCKER] Multi-turn continuous-context behavior is not tested.
   - Evidence: every exported row starts from a fresh Python session (`runtime/export_session_bundle.py:98-101`), finalizes once (`runtime/export_session_bundle.py:103-127`), and returns one independent row. C++ then resets the native session at the start of every row (`runtime/cpp/session_main.cpp:968-969`). Production's speculative finish explicitly retains ASR context after finalize (`src/nemotron_speech/server.py:9014-9035`); cold reset is only for true boundary reasons such as `close` or `end` (`src/nemotron_speech/server.py:94`, `src/nemotron_speech/server.py:9079-9123`).
   - Miss: the most production-relevant continuous path is "finalize, keep caches/decoder/text collector, receive more audio, emit next steady and next final." The C++ gate never appends turn B to the same session after turn A's speculative finalize. finalize_ref has a reset/resume test for this (`runtime/finalize_ref.py:1158-1202`), but the bundle does not carry it into C++.
   - Concrete fix: export multi-turn scripts: turn A audio, speculative finalize event, turn B continuation in the same session, next steady event(s), second speculative finalize, then true-boundary cold reset. Assert retained caches, collector text, deltas, and final cumulative tokens across the whole session.

8. [MAJOR] The oracle chain is transitive and weak: C++ == `finalize_ref`, while `finalize_ref` is only a proxy for production.
   - Evidence: `finalize_ref` states it "mirrors the production continuous-finalize path" (`runtime/finalize_ref.py:2-13`), but its NeMo canary set is four indices (`runtime/finalize_ref.py:45`) and the canary loop checks token exactness against NeMo chunked stream+finalize (`runtime/finalize_ref.py:1416-1429`). C++ compares against bundle `gold_tokens` produced by that reference (`runtime/export_session_bundle.py:127-145`, `runtime/cpp/session_main.cpp:877-880`).
   - Production is not just the synchronous reference: it includes websocket sends (`src/nemotron_speech/server.py:8966-8979`), send-failure behavior (`src/nemotron_speech/server.py:8989-8997`), scheduler generations (`src/nemotron_speech/server.py:8694-8729`), and continuous/cold-reset branching (`src/nemotron_speech/server.py:9014-9131`).
   - Miss: "behavioral equivalence to Python" currently means equivalence to an internal synchronous proxy, not equivalence to the shipping server's emitted output.
   - Concrete fix: add a production-server oracle run for the same audio scripts and compare emitted JSON event streams. Keep `finalize_ref` as a unit-level executable spec, but do not let it be the only T1 oracle.

9. [MAJOR] The gate does not implement the full-1000 WER and per-utterance WER parts of T1.
   - Evidence: the formal bar requires full-1000 WER non-inferiority and per-utterance WER bounded (`reviews/0.4-decision-FINAL.md:23-26`). The C++ harness accepts however many rows are in `session_bundle.ts` and reports only row pass counts (`runtime/cpp/session_main.cpp:962`, `runtime/cpp/session_main.cpp:1011-1016`). The exporter default is `--n 20`, not 1000 (`runtime/export_session_bundle.py:230-234`), and there is no reference text or WER calculation in the bundle schema (`runtime/export_session_bundle.py:190-224`).
   - Miss: 200/200 token exactness on a selected bundle is not a corpus WER gate, not a CI-bound non-inferiority test, and not a per-utterance WER guard.
   - Concrete fix: make the C++/server e2e run all 1000 benchmark utterances with references, compute the agreed WER metric, enforce CI and per-utterance bounds, and publish the exact sample list/normalizer/version with the gate result.

10. [MAJOR] The first chunk is still TorchScript, so "pure native runtime" is not what 1.4 proves.
    - Evidence: C++ loads `enc_first.ts` as a TorchScript module (`runtime/cpp/session_main.cpp:950-952`) and calls it through `enc_first.forward(...)` for the first steady chunk (`runtime/cpp/session_main.cpp:784-795`, `runtime/cpp/session_main.cpp:839-843`). Only later steady chunks use `AOTIModelPackageLoader` (`runtime/cpp/session_main.cpp:798-812`, `runtime/cpp/session_main.cpp:848-850`).
    - The encoder export script traces first and steady TorchScript graphs (`runtime/export_stream_encoder.py:48-56`) and documents traced-stream encoder drift around `~1e-5` while remaining token-exact (`runtime/export_stream_encoder.py:84-86`).
    - Miss: token exactness here is an argmax-margin observation over this corpus/bundle, not proof that the first chunk is native, byte-exact, or safe under near-ties.
    - Concrete fix: compile/export a first-chunk native/AOTI path or keep TorchScript explicitly out of the "native" claim. Add first-chunk numeric drift, argmax-margin, and token divergence reporting to the gate.

11. [MAJOR] FORK_ASSERT timing is meaningful, but its scope is too narrow and the per-row reset masks lifecycle bugs.
    - Evidence: C++ snapshots parent ASR state, flips parent mode, clones a fork, mutates the fork through finalize AOTI/decode, and then asserts parent tensors/tokens unchanged (`runtime/cpp/session_main.cpp:873-875`, `runtime/cpp/session_main.cpp:895-915`). So the assert does run after fork mutation. The problem is that mode is not part of the snapshot, parent mode is intentionally changed before cloning (`runtime/cpp/session_main.cpp:873-875`), then restored (`runtime/cpp/session_main.cpp:916`), and the whole session is discarded on the next row (`runtime/cpp/session_main.cpp:968-969`).
    - finalize_ref snapshots a broader parent state including pending audio, raw ring, mel ring, emitted frames, and total audio samples (`runtime/finalize_ref.py:599-612`). Production's fork assert is centered on cache tensors and hypotheses/prediction state (`src/nemotron_speech/server.py:7607-7668`), but production also has retained-context lifecycle that the C++ reset never exercises.
    - Miss: a bug that leaves the parent in the wrong lifecycle state after speculative finalize, corrupts audio/ring state, or only breaks when more audio arrives after finalize can pass because the next C++ iteration starts from `init_*`.
    - Concrete fix: assert `mode`/continuous state transitions, audio/ring counters, and post-finalize continuation behavior in one long-lived C++ session. Do not reset between turns for the continuous-context test.

12. [MAJOR] AOTI output ownership and recurrent-cache aliasing are assumed, not proven.
    - Evidence: steady encoder outputs are installed directly as recurrent session caches (`runtime/cpp/session_main.cpp:772-781`) and then passed back into AOTI on the next chunk after `.contiguous()` (`runtime/cpp/session_main.cpp:803-810`). The fork clone deep-copies the current session tensors (`runtime/cpp/session_main.cpp:206-218`), but there is no assertion that AOTI outputs do not alias inputs or loader-owned reusable storage before becoming long-lived session state.
    - Miss: if the loader/runtime reuses output buffers or mutates an input-like output in a later call, the harness may only notice when tokens flip. The current FORK_ASSERT covers parent-vs-fork after finalize, not steady-output ownership across a long stream.
    - Concrete fix: either clone cache outputs on assignment or add explicit storage-alias assertions between inputs/outputs and across consecutive `loader.run()` calls. Add a long-stream cache stability check independent of token equality.

13. [MINOR] The `final_T == 0` path has less geometry validation than non-empty finalize.
    - Evidence: when the exporter has no finalize inputs, it writes `final_drop_extra = -1` and `final_T = 0` (`runtime/export_session_bundle.py:107-118`). C++ skips the entire finalize geometry block when `final_T == 0` and only compares cumulative tokens (`runtime/cpp/session_main.cpp:881-915`).
    - Miss: a no-pending-audio final can hide wrong `drop_extra` metadata or state-transition handling. Production explicitly returns current text when there is no pending audio (`src/nemotron_speech/server.py:9874-9877`) and still goes through emit/suppression logic afterward (`src/nemotron_speech/server.py:8936-9012`).
    - Concrete fix: add explicit zero-finalize cases with expected metadata and event outcome: no model call, correct delta/suppression, correct retained/cold lifecycle.

14. [MINOR] The row-count result is not an adversarial coverage statement.
    - Evidence: the C++ harness reports `steady=N/rows final_token_exact=N/rows FORK_ASSERT=N/rows` (`runtime/cpp/session_main.cpp:1011-1016`). The bundle rows are sequential `start + offset` samples (`runtime/export_session_bundle.py:245-250`), with no coverage stratification by short/no-steady, exact-boundary residuals, long streams, high token rate, correction cases, duplicate finals, or generation races.
    - Miss: "200/200" can be a narrow happy-path sample count. It does not quantify coverage of the failure modes that distinguish server behavior from cumulative token replay.
    - Concrete fix: publish a coverage manifest: residual-frame buckets, zero/one/many steady chunks, long streams, correction/non-prefix deltas, no-pending finalize, duplicate finalize, vad_start cancellation, reset/resume, stale generation, and true-boundary cold reset.

Top 5:
1. Event-stream equivalence is missing; 1.4 checks cumulative tokens only.
2. Final-delta and duplicate/empty-final suppression are missing.
3. Generation suppression/stale-final behavior is missing.
4. The harness is mel-fed, so C++ audio preprocessing and final remainder assembly are unvalidated.
5. Multi-turn continuous context retention after speculative finalize is not exercised.

Verdict: MET-WITH-CAVEATS only if the claim is narrowed to "C++ mel-level cumulative steady/final tokens match `finalize_ref` on the exported bundle, and forked finalize does not mutate the checked parent tensors." It is not met for "Phase-1 single-stream T1 behavioral equivalence" as defined in the decision memo. Caveats that block the stronger claim: no exact event stream, no production final-delta comparison, no generation-suppression cases, no audio-fed C++ preprocessing/remainder math, no multi-turn retained-context session, no full-1000 WER/per-utterance WER enforcement, and a TorchScript first chunk in the supposed native runtime.
