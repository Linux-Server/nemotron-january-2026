# Codex adversarial review - 1.4b Step 1 event stream

1. [BLOCKER] Token-level delta is not equivalent to the word-level reference.

   Ground truth is explicitly word-based: `finalize_ref._continuous_append_only_delta` calls `final_text.split()` and `emitted_text.split()` before common-prefix, suppression, and overlap trimming (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/finalize_ref.py:144`; same server helper at `/home/khkramer/src/nemotron-january-2026/src/nemotron_speech/server.py:392`). The C++ implementation ports that logic over raw token ids instead (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:180`), and final emission consumes that token delta directly (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1071`).

   That is not a valid general equivalence. Constructed subword case: emitted text is `I live in New`, with ids like `[I, live, in, _New]`; final text is `I live in Newark`, with ids like `[I, live, in, _New, ark]`. The Python/server word algorithm sees the same word count with a non-prefix correction at the last word and suppresses (`len(final_words) <= len(emitted_words)`, `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/finalize_ref.py:157`). The C++ token algorithm sees all emitted ids as a prefix and emits `[ark]` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:191`). That mutates an already-emitted word, exactly what the word-level append-only collector is trying to prevent.

   The exporter only proves the token proxy for the observed bundle events, not for the runtime contract. It computes the same token-level proxy (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:81`), then checks that decoded proxy events match the word delta for those rows (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:218`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:238`). That is corpus validation, not a proof. Fix: either implement final-delta on decoded text/words in the C++ runtime, or export enough tokenizer word-boundary metadata to compute the same word-level decision in C++. Add explicit adversarial cases where a prior final ends inside a later-corrected word (`New` -> `Newark`, `play` -> `playing`) and require C++ to match the word delta, not merely the token proxy.

2. [BLOCKER] The exported text oracle is dead in the C++ assertion.

   The file comment says text gold is exported but C++ compares token-id payloads (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:6`). That is exactly what happens: `gold_events_from_bundle` reads only `event_kinds`, `event_tokens`, and `event_collector_tokens` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:244`). The exporter writes `event_text_bytes` and `event_collector_text_bytes` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:377`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:390`), but no C++ code reads them.

   This means the assertion can never catch the word-vs-token failure above unless the exact corpus row happens to trigger the export-time proxy assertion. Fix: load and compare text event payloads in the C++ event assertion by making the runtime emit text, or add a tokenizer/text decoder to the harness and compare decoded C++ events to `event_text_bytes` and `event_collector_text_bytes`. If the production runtime is intentionally token-only, the claim must be narrowed: it is token-proxy equivalence on this bundle, not server text-delta equivalence.

3. [BLOCKER] The collector/suppression edge cases are mostly untested because every bundle row has a fresh session and one finalization.

   The exporter creates a new session per sample (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:254`), calls one `vad_stop`, then one `debounce_expire` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:259`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:283`). The C++ replay also resets the session for every row (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1144`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1145`). Therefore `continuous_emitted_tokens` is empty before the final event in normal non-empty rows.

   That setup cannot validate duplicate final suppression, shortened final suppression, non-prefix correction handling, or suffix/prefix overlap trimming against a non-empty collector. The C++ branches exist (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:191`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:194`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:197`), but the 20/20 and 199/200 event results mostly exercise "collector empty, append all final tokens." Fix: add deterministic unit rows or synthetic bundle rows with pre-populated emitted collector state, and add true multi-final continuous sessions without resetting between utterances. Required cases: duplicate final, final shorter than collector, non-prefix correction with longer final, and overlap trim.

4. [BLOCKER] "Generation suppression equivalence" is not modeled.

   The production server suppresses stale finals when `expected_generation` no longer matches `session.scheduler_generation`, and returns before updating committed/last/collector state (`/home/khkramer/src/nemotron-january-2026/src/nemotron_speech/server.py:8941`; `/home/khkramer/src/nemotron-january-2026/src/nemotron_speech/server.py:8959`). The C++ final path has no generation fields and only suppresses when the append-only delta is empty (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1071`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1073`). The exporter also records only the finalize_ref empty/non-empty result path (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:217`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:230`).

   If Step 1 claims generation-suppression equivalence, this is a hard miss. Fix: either remove "generation" from the Step 1 claim, or add scheduler generation to the replay state and export stale-final scenarios that assert no final event/collector update occurs when the generation is stale.

5. [MAJOR] Exact interim event-stream comparison uses the wrong oracle for the production runtime.

   Gold interim events come from `RecordingContinuousFinalizeRef._process_one_steady_chunk`, which calls the eager Python encoder via `super()` and records text changes (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:188`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:190`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:200`). C++ continuation chunks use `enc_steady_aoti.pt2` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:999`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1000`) and emit interim events when token ids change (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1008`).

   If AOTI drift moves a near-threshold token one chunk earlier, an exact eager-vs-AOTI interim stream failure is expected and does not isolate event logic. The reported N=200 199/200 is therefore not a clean correctness signal. Fix: split the gates. For event logic, generate the oracle from the same AOTI steady hypotheses the C++ runtime uses, or feed captured AOTI per-chunk token sequences through a pure Python collector. Keep eager-vs-AOTI equivalence as a separate numerical/model parity gate. If Step 1 includes exact interim ordering, the current 199/200 means the bar is not met; if Step 1 is only final delta/suppression, exclude interims from the exact event gate and document the tolerance.

6. [MAJOR] Interim emission is token-change based in C++, but text-change based in the server/export oracle.

   The server emits an interim only when decoded `text != session.current_text` (`/home/khkramer/src/nemotron-january-2026/src/nemotron_speech/server.py:9165`; `/home/khkramer/src/nemotron-january-2026/src/nemotron_speech/server.py:9168`). The exporter mirrors this and only records when `text_changed` is true (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:191`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:200`). C++ emits on raw token-vector change (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:1008`).

   The exporter asserts token/text change equivalence for observed chunks (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/export_session_bundle.py:193`), but that is again corpus validation. Subword tokenizers can have multiple segmentations for the same surface text; if RNNT ever changes tokenization without changing decoded text, C++ will emit an extra interim that the server would suppress. Fix: make C++ interim emission text-based, or export tokenizer-normalized text-change markers/word-boundary metadata and assert against those. At minimum, add a targeted test where token ids change but decoded text is unchanged.

7. [MINOR] The ordered stream assertion itself is strict, but only for token fields.

   `equal_events` does check event count and then compares each event by index (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:277`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:283`). It compares `kind`, `tokens`, and `collector_tokens` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:284`). So a different count or non-identical interleaving will not pass, except for indistinguishable duplicate events. The missing part is text semantics, not ordering. Fix: keep this strict ordered assertion, but extend the compared payload to the word-level text delta/collector or decoded text.

8. [MINOR] Low direct regression risk to token and FORK_ASSERT checks, but the new event state is outside FORK_ASSERT coverage.

   Event tracking does not feed encoder/decoder inputs, so it is unlikely to perturb `steady_tokens` or final cumulative tokens. The ASR fork assert still checks cache tensors, decoder state, emitted frames, and `hyp` (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:393`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:398`). But `last_interim_tokens` and `continuous_emitted_tokens` are copied/reset (`/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:364`; `/home/khkramer/src/nemotron-january-2026/proj-2026-05-24-from-scratch-runtime/runtime/cpp/session_main.cpp:416`) and are not part of the fork assert. That is reasonable for ASR-state immutability, but do not read `FORK_ASSERT=PASS` as proving event collector state is correct. Fix: add separate event-state invariants after finalize: suppressed events do not mutate collector, final events append exactly one delta, and collector text/token state matches gold after each event.

## Bottom line

Step 1's claimed bar, as written ("event/delta/suppression equivalence"), is **not met**. What is currently demonstrated is narrower: full ordered token-proxy event equality on N=20, and almost-full equality on N=200 against an eager interim oracle, with final cumulative tokens exact.

The word-vs-token delta is a **real correctness risk**, not a theoretical nit. The reference/server contract is word-level append-only text. The C++ implementation makes suppression decisions at token-id granularity, the text gold is not consumed by the C++ assertion, and the fixture does not exercise non-empty collector correction cases where subword boundaries matter.
