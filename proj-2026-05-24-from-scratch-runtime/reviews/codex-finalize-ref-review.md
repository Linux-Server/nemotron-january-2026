# Adversarial review: finalize_ref/export_finalize_encoder

1. [BLOCKER] The validation oracle is the wrong algorithm. `runtime/finalize_ref.py:857-864` defines `full_greedy_tokens()` as `model.forward(whole audio) + ref_greedy`, and the gate accepts `result.final_tokens == full_tokens` or normalized text equality at `runtime/finalize_ref.py:923-932`. That is a single offline encoder pass, not the production chunked cache-aware stream plus a keep-all finalize fork. Production steady chunks are `keep_all_outputs=False` with threaded caches (`src/nemotron_speech/server.py:9255-9268`, `src/nemotron_speech/server.py:9676-9686`), and finalize is a separate `keep_all_outputs=True` remainder pass (`src/nemotron_speech/server.py:8009-8082`, `src/nemotron_speech/server.py:9988-9999`). This gate can report a "bug" when streaming differs from offline, or pass when the finalize path diverges from production but happens to normalize to the same text. Fix: add a NeMo chunked streaming oracle: run the same normal chunk loop with `encoder.cache_aware_stream_step(... keep_all_outputs=False)`, then the same finalize fork/remainder with `keep_all_outputs=True`, decoding each encoder output with `model.decoding.rnnt_decoder_predictions_tensor(..., partial_hypotheses=nemo_prev)` as in `runtime/stream_decode.py:59-70`. Require token-exact equality against that oracle; keep offline full-greedy only as a diagnostic.

2. [BLOCKER] Speculative finalize copies the disposable fork back into the live session. `_finish_speculative()` calls `_copy_finalized_fork_state()` at `runtime/finalize_ref.py:817-824`, and that copy overwrites live caches, decoder state, pred output, mel ring, emitted frame count, raw ring, pending audio, total samples, tokens, and current text at `runtime/finalize_ref.py:799-815`. Production does not do this. `_continuous_finish_speculative_finalize_locked()` only clears debounce bookkeeping and returns to `STREAMING` at `src/nemotron_speech/server.py:9014-9027`; the fork is built as disposable state (`src/nemotron_speech/server.py:7473-7528`) and the parent is asserted unchanged after flush (`src/nemotron_speech/server.py:7896-7901`, `src/nemotron_speech/server.py:7607-7668`). The reference therefore injects final-padding silence and keep-all-output right-context into the next live chunk. Under production, the next chunk's left context is the pre-finalize parent cache/mel ring plus real pending/post-stop audio. Under this reference, the next chunk starts from the fork's post-finalize cache, a mel/raw ring contaminated by appended silence, `pending_audio=[]`, and `emitted_frames` advanced through the synthetic tail. Fix: delete `_copy_finalized_fork_state()` from speculative finish; `_finish_speculative()` should only set `state=STREAMING`, clear debounce/reset flags, and leave ASR state untouched. Also update `runtime/1.3-finalize-spec.md:41-43`, which currently contradicts the server authority by implying fork-final state can be retained.

3. [BLOCKER] Post-stop audio can be stranded after debounce finalize. While `PENDING_FINALIZE`, `append_audio()` stores audio in `continuous_post_stop_audio` and returns at `runtime/finalize_ref.py:373-377`. After debounce, `_finish_speculative()` sets the session back to `STREAMING` at `runtime/finalize_ref.py:824`, but the next `append_audio()` path at `runtime/finalize_ref.py:369-385` never flushes already-held `continuous_post_stop_audio` before appending new audio. Production explicitly flushes held post-stop bytes on streaming resume before appending the new frame (`src/nemotron_speech/server.py:6688-6709`; same non-scheduler logic at `src/nemotron_speech/server.py:7202-7227`). With the current reference, audio that arrived during the debounce window remains in the side buffer until a later `vad_start()` happens, and ordinary audio frames after speculative finish are processed without it. Fix: when `append_audio()` sees `STREAMING` and `continuous_post_stop_audio` is nonempty, move held audio into `pending_audio`, update `total_audio_samples`, drain if ready, then append the new audio. Add a test where audio arrives during debounce, debounce finalizes, and streaming resumes without an explicit `vad_start`.

4. [MAJOR] FORK_ASSERT is too narrow and is checked before the reference's most destructive mutation. The reference snapshot covers only cache tensors, `decoder_state`, `pred_out_stream`, and `hyp_tokens` at `runtime/finalize_ref.py:572-596`. It does not cover `pending_audio`, `raw_audio_ring`, `mel_frame_ring`, `emitted_frames`, `total_audio_samples`, `synthetic_prefix_samples`, `current_text`, or `continuous_post_stop_audio`, even though those fields are part of the live streaming state and several are overwritten by `_copy_finalized_fork_state()` at `runtime/finalize_ref.py:804-815`. Production's FORK_ASSERT covers the core cache/hypothesis/pred state (`src/nemotron_speech/server.py:7607-7668`), but the reference needs a stronger assertion because it is serving as an executable spec for native state ownership. Fix: snapshot and assert every parent field that must be unchanged by fork flushing, then assert again after speculative finish that only the allowed debounce/text bookkeeping changed. At minimum, assert that no fork-owned `pending_audio`, raw ring, mel ring, emitted count, or cache object is copied into the parent on speculative finish.

5. [MAJOR] The exporter can manufacture a first-finalize geometry that production cannot reach. For `kind == "first"`, `_make_row()` bypasses the streaming path and assigns the entire wav to a fresh session at `runtime/export_finalize_encoder.py:93-97`, leaving `emitted_frames == 0` no matter how long the sample is. Production drains ready chunks before processing non-audio events (`src/nemotron_speech/server.py:6666-6672`) and uses the ready predicate at `src/nemotron_speech/server.py:5826-5838`, so a long utterance cannot remain a first finalize with `drop_extra=0`; it would already have steady chunks and a `mel_frame_ring`. Fix: for first-finalize fixtures, assert the sample is shorter than `preprocess_new_audio_samples` or synthesize/clip a short sample that cannot trigger a steady chunk. For long samples, build fixtures only by feeding audio through `append_audio()` / production event ordering.

6. [MAJOR] Exported token "gold" is self-referential, not production gold. `_make_row()` saves `gold_final_tokens` from the same `flush_finalize_fork()` implementation under review and also saves `full_greedy_tokens` from the wrong offline oracle at `runtime/export_finalize_encoder.py:105-132`. There is no comparison against NeMo's chunked streaming+finalize decoder, and `main()` returns success unconditionally at `runtime/export_finalize_encoder.py:261-266` even if the algorithmic gold is wrong. This can give a C++ port a fixture suite that only proves it copied Python's bug. Fix: have the exporter compute and save `nemo_stream_finalize_tokens` from the production-shaped oracle described above, fail if `gold_final_tokens` differs, and rename offline tokens to `offline_full_greedy_tokens` so they are not treated as an oracle.

7. [MAJOR] The reset/resume canary is too weak to catch the fork-copy pollution. The speculative reset test only checks state, nonempty deltas, and normalized delta text against offline full-greedy for two turns at `runtime/finalize_ref.py:958-978`. That will not catch the live cache/mel-ring/pending-audio corruption from `runtime/finalize_ref.py:799-824`, because the append-only delta can still equal the second turn's normalized text even when the live stream starts from synthetic final-padding state. Fix: snapshot live ASR state immediately before speculative finalize and assert cache/decoder/mel/raw/pending/emitted fields are identical after `_finish_speculative()`. Then append a continuation segment and compare the next steady chunk plus next finalize against the NeMo chunked oracle, not against offline per-turn text.

8. [MAJOR] There is no `vad_start` cancellation test. The reference implements `vad_start()` cancellation and held-audio flushing at `runtime/finalize_ref.py:399-408`, but `validation_gate()` never calls it (`runtime/finalize_ref.py:900-1009`). Production's cancellation path is explicit: cancel debounce, return to `STREAMING`, clear timestamps, and flush post-stop audio back into the stream at `src/nemotron_speech/server.py:6830-6847`. This is the path that proves a speculative fork can be discarded without mutating the parent. Fix: add a test that calls `vad_stop()`, appends post-stop audio, calls `vad_start()` before debounce expiry, asserts no final was emitted, asserts held audio was flushed, and compares the resulting continuous stream to a no-stop chunked oracle.

9. [MAJOR] Remainder-frame math is only spot-checked by incidental dataset lengths. `prepare_finalize_inputs()` contains the fragile finalize math at `runtime/finalize_ref.py:638-691`: `remaining_frames`, the multi-call fixed-preprocessor loop, the partial final iteration, `drop_extra` selection, and mel-ring prepend. The line-by-line formula matches the cited server shape (`src/nemotron_speech/server.py:8009-8082`, `src/nemotron_speech/server.py:9881-9999`), but the gate exercises only four fixed canary indices (`runtime/finalize_ref.py:45`, `runtime/finalize_ref.py:919-949`) and does not enumerate residual lengths. An off-by-one in the last partial call, raw-ring advance, or pre-encode prepend can survive the current tests if the chosen samples do not hit the boundary. Fix: synthesize or clip audio to cover every residual frame count around `0..shift_frames`, both `emitted_frames==0` and continuation, exact first-ready threshold, exact full-shift final iteration, and one-sample-below/above boundaries. Assert `remaining_frames`, `chunk_mel.shape[-1]`, `drop_extra`, and token output against the NeMo chunked oracle for each case.

10. [MAJOR] The reference clone/assert helpers are weaker than production for NeMo state. `clone_tree()` in the reference falls through to `copy.deepcopy()` and returns the original object on failure at `runtime/finalize_ref.py:89-92`; `assert_tree_equal()` has no branch for generic NeMo objects at `runtime/finalize_ref.py:130-141`. Production explicitly clones NeMo `__dict__` objects (`src/nemotron_speech/server.py:197-204`), deep-clones previous hypotheses (`src/nemotron_speech/server.py:212-216`), and asserts NeMo object fields recursively (`src/nemotron_speech/server.py:373-382`). This matters as soon as the correct NeMo `partial_hypotheses` oracle is added, and it matters for native-port ownership tests. Fix: port production's NeMo-object clone/assert branches and `clone_hypotheses_deep()` into the reference, or make the reference fail loudly if it sees a state object it cannot clone byte-independently.

11. [MAJOR] The TorchScript trace check is not a dynamic finalize-shape proof. `export_finalize_encoder.py` traces one example per `drop_extra` with `check_trace=False` at `runtime/export_finalize_encoder.py:193-208`, then tests only the small row set built at `runtime/export_finalize_encoder.py:163-168`. Finalize `T` is dynamic by construction, but the script can print `PASS byte-exact trace` at `runtime/export_finalize_encoder.py:261-264` without proving arbitrary residual `T` values or future buckets. `_tensor_outputs_equal()` also zips outputs at `runtime/export_finalize_encoder.py:75-85` without checking tuple length, so missing/extra outputs would be ignored. Fix: check output arity and metadata first, run `check_trace` or explicit replay over an exhaustive residual-frame grid, and prefer `torch.export`/AOTI or the native encoder step for the C++ path. Add `--require-trace` that exits nonzero if trace coverage is not proven.

12. [MINOR] Geometry and fixed-FFT assumptions are printed, not gated. The reference derives the fixed preprocessor plan at `runtime/finalize_ref.py:286-333` and builds fixed audio at `runtime/finalize_ref.py:439-490`, mirroring server derivation at `src/nemotron_speech/server.py:1584-1639` and fixed preprocessing at `src/nemotron_speech/server.py:4518-4567`. The gate only prints the derived values at `runtime/finalize_ref.py:906-912`. It does not assert the production constants under test, and it does not perform mel-level checks that fixed-plan slices match the intended timeline for boundary lengths. Fix: assert `shift_frames`, `pre_encode_cache_size`, `drop_extra`, `final_padding_frames`, `first_preprocess_mel_frame`, and `constant_preprocess_samples` against the server configuration being targeted; add mel-slice hash/equality tests for first, steady, and final partial windows.

13. [MINOR] The reference hard-codes runtime mode that production can configure. `RIGHT_CONTEXT=1` and `FINALIZE_SILENCE_MS=150` are fixed at `runtime/finalize_ref.py:43-45`, and `load_model()` pins `att_context_size=[70, 1]` plus `greedy_batch` decoding at `runtime/finalize_ref.py:189-200`. Production selects attention context and final padding from server configuration (`src/nemotron_speech/server.py:1503-1513`, `src/nemotron_speech/server.py:1603-1608`) and can use `greedy` by default or `greedy_batch` under batching (`src/nemotron_speech/server.py:1515-1561`). Fix: either parameterize the reference/exporter from the same server config/env and record it in fixtures, or assert that the current run is intentionally pinned to the exact production mode being ported.

## Top 5

1. Replace the offline full-greedy validation/export oracle with NeMo's chunked streaming+finalize oracle.
2. Remove the speculative fork-to-parent copy; the live parent ASR state must remain pre-finalize.
3. Flush `continuous_post_stop_audio` on streaming resume after debounce, not only on `vad_start`.
4. Strengthen reset/resume tests to assert live state invariants and next-chunk behavior, not just normalized deltas.
5. Fix exporter fixtures so "gold" means production oracle output and first-finalize rows are production-reachable.
