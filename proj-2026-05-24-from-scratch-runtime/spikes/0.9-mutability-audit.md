# Spike 0.9 — Model mutability audit (COMPLETE)

**Goal (PLAN §6 / 0.9):** enumerate every mutable model surface touched per request, so the native design can prove it
passes them as **pure call parameters** with **zero shared mutable state on the hot path** — the prerequisite for the
**shared-read-only-weights** density lever. Today's per-lane *full model replicas* (`server.py:3109-3137`) dodge these
races at a large memory cost; sharing one weight copy across lane threads is only safe if every surface below is
parameterized.

This audit is pure code-archaeology (no GPU needed) and is **complete**. Each surface is tagged:
- **PARAM** — must become an explicit per-call argument in the native runtime (no shared state).
- **PER-LANE** — must be per-lane/per-thread instance state (not shared), or eliminated.
- **IMMUTABLE** — safe to share read-only.

## Mutable surfaces (per-request) found in source

| # | Surface | Where (anchor) | Today's behavior | Native requirement |
|---|---|---|---|---|
| 1 | `encoder.streaming_cfg.drop_extra_pre_encoded` | `cudagraph_encoder.py:56-64` (save/restore wrapper); also touched in batched path `batch_primitives.py:142-149` | **mutated on the model object** before the encoder call and restored in `finally` — proves NeMo reads it from shared module config, not an arg. First chunk uses `0`, steady uses `2`, finalize differs. | **PARAM** — pass `drop_extra` explicitly into the native encoder; never mutate a shared field. Two lanes running first-chunk + steady concurrently on a shared object would race here. |
| 2 | Prompt / target-language state (`set_inference_prompt`) | `server.py:1288-1292`, `:8306-8307`, `:7394-7395` | prompted models mutate the model's prompt state per session/chunk before inference | **PARAM** (or **PER-LANE**) — prompt conditioning must be a per-request input. (v1 is EN-only, so this is out of v1 scope, but the native API must not bake a shared prompt slot.) |
| 3 | Decoder/joint `eval()`↔`train()` toggles | NeMo greedy forwards: `rnnt_greedy_decoding.py:390-415`, `:744-766` | NeMo temporarily switches decoder/joint to eval then restores training state around the decode | **IMMUTABLE-by-construction** — the native decode runs inference-only; never toggles a shared training flag. Ensure no global `training` bool is flipped per call. |
| 4 | CUDA-graph static input/output buffers | `cudagraph_encoder.py:193-210` (alloc), `:272-281` (replay copies in, returns static output pool), `:15-17` (caller must clone before next replay) | each captured bucket owns static buffers; replay overwrites them | **PER-LANE** — static buffers are NOT shareable across concurrent replays; see `0.11-graph-ownership.md`. Outputs must be cloned before the next replay. |
| 5 | Featurizer `dither` | `server.py:1510`, per-lane set `:3139` | set to `0.0` once at load (not per-request) | **IMMUTABLE** if set once at construction; ensure the native preprocessor fixes dither=0 at build, not per call. |
| 6 | Preprocessor cuFFT plan / STFT state | `server.py:1509-1593` (constant-plan) | constant plan held to keep cuFFT deterministic | **PER-LANE or IMMUTABLE** — a constant plan can be shared read-only IF no per-call plan re-selection occurs; validate in 0.8. Memory `cufft-stft-plan-size-nondeterminism`. |
| 7 | RNG / determinism flags (TF32, cuBLAS workspace, deterministic algos) | TF32 disabled under batching `server.py:655-660` | process-global torch flags | **PER-PROCESS pinned at startup** (PLAN T0 contract): TF32 off, deterministic algos, pinned cuBLAS workspace, **per-lane cuBLAS/cuDNN handles** — these are not per-call but must be locked so shared weights → identical outputs across lanes. |
| 8 | Decoder hypothesis / `LabelLoopingStateItem` (per-session) | `batch_primitives.py:109-126` (alias guards), clone helpers `server.py:147-207,249-263`, fork deep-clone `:6370-6425` | per-session, alias-sensitive (NeMo mutates hyps in place) | **PER-SESSION state**, never shared; the native state-ownership model (PLAN 1.0) owns this. NOT a model-weights surface, but listed because aliasing here is the classic concurrency bug. |

## Conclusion / design rule for the native runtime
**Shared read-only weights are viable ONLY IF:** (a) surfaces #1, #2 become pure call parameters; (b) #4 buffers are
per-lane (graph-ownership spike 0.11); (c) #3 never flips a shared training flag; (d) #7 determinism flags are pinned
per-process with per-lane handles; (e) #6 cuFFT plan is build-time-constant. If any of these can't be parameterized,
**fall back to per-lane replicas** (today's behavior) → the shared-weights density win is forfeit → re-run the §0
worth-it gate (per the PLAN 0.4 decision tree, "0.9 fails" row).

**No source was modified by this audit.**
