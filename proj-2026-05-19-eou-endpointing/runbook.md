CLI subsetting verdict: the current `stt-benchmark run` command has `--limit`, but no `--ids`,

below runs the normal 1000-sample benchmark and captures snapshots for all sessions; downstream

## Start the server

```bash
cd /home/khkramer/src/nemotron-january-2026
mkdir -p ./eou-collect/snapshots ./eou-collect/telemetry ./eou-collect/logs

env \
  NEMOTRON_EOU_PROBE=1 \
  NEMOTRON_EOU_SNAPSHOT_DIR=./eou-collect/snapshots \
  NEMOTRON_EOU_SNAPSHOT_EVERY=1 \
  NEMOTRON_EOU_CLIENT=1 \
  NEMOTRON_CONTINUOUS=1 \
  NEMOTRON_FINALIZE_SILENCE_MS=150 \
  NEMOTRON_FORK_ASSERT=1 \
  NEMOTRON_RUN_TAG=eou_step2_collect \
  NEMOTRON_TELEMETRY_DIR=./eou-collect/telemetry \
  /home/khkramer/src/nemotron-nano-omni/.venv-asr/bin/python \
    src/nemotron_speech/server.py \
    --host 127.0.0.1 \
    --port 8080 \
    --right-context 1 \
  2>&1 | tee ./eou-collect/logs/eou_step2_collect.server.log
```
