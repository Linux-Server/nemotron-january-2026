NEW FLAG: `NEMOTRON_MODEL_LANES` (int, default 1). 1 = TODAY'S EXACT behavior (single lane, byte-identical,
rollback-safe). >1 (prototype N=2) = up to N model calls in flight concurrently, each on its own
`torch.cuda.Stream` + its own worker thread. Only meaningful with the scheduler/batching on
(`NEMOTRON_SCHEDULER_B1=1`); if lanes>1 but scheduler off, log + fall back to lanes=1.

DESIGN (conservative, correctness first):
- Today the scheduler forms a ready-batch and runs ONE `conformer_stream_step(B)` at a time (the serial lane).
  Change: allow up to N such batches to be DISPATCHED concurrently to N lanes (worker threads), each lane with
  its own CUDA stream; the scheduler keeps forming batches and hands each to a free lane.
- HARD HAZARD #1 — the GLOBAL `encoder.streaming_cfg.drop_extra_pre_encoded`. NeMo mutates this shared field
  per call (see `conformer_stream_step_restoring_drop_extra` in batch_primitives.py). Two lanes with DIFFERENT
  drop_extra racing on the global = corruption. GUARD: steady-state chunks all share drop_extra=2, so concurrent
  STEADY batches are safe; route any NON-steady call (first-chunk warmup, the vad_stop finalize/barrier-drain,
  any drop_extra≠steady) to a SERIALIZED exclusive path (lane 0 only, no other lane in flight). Simplest correct
  rule: only batches with the SAME drop_extra may run concurrently; otherwise serialize. Document the rule you
  pick.
- HARD HAZARD #2 — a session must NEVER be in two lanes at once (its cache/hyps would corrupt). The scheduler
  must not dispatch a session to lane B while it's still in flight on lane A. Track in-flight session IDs;
  hold a session out of batch-forming until its lane completes + scatters.
- HARD HAZARD #3 — CUDA-stream correctness: each lane runs on its own stream; the scatter / state read-back
  must `stream.synchronize()` (or event-wait) BEFORE copying state back to the session, or you read stale GPU
  memory. The owned-clone scatter stays.
- Keep `inference_lock` semantics where they protect shared mutable model state; the point is to overlap the
  HOST dispatch of independent lanes, not to remove necessary synchronization. Per-session state_lock + the
  generation-token guard still apply.

  (1) BYTE-EXACT: with NEMOTRON_MODEL_LANES=2 + scheduler/batching on, the per-stream transcripts must be
